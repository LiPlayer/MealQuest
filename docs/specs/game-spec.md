# 餐餐有戏 - 游戏化全景需求 (Master Game Spec V1.0)

> **关联文档**: [餐餐有戏 - 产品全景定义白皮书](./mealquest-spec.md)

## 1. 核心定位
本游戏模块旨在通过轻量级的互动娱乐，增加用户在候餐期间的粘性，并产出“碎银”与“碎片”资产，形成商业闭环。

## 2. 游戏列表 (Phase 1)

### 2.1 神厨飞刀 (Blade Chef)
*   **玩法原型**: 类似于《见缝插针》或《水果忍者》的变体。
*   **核心循环**: 投掷飞刀击中旋转的食材 -> 获得积分 -> 积分兑换碎银。
*   **资产产出**:
    *   **常规产出**: 碎银 (Soft Currency)
    *   **稀有掉落**: 食材碎片 (Fragments) - 受每日风控限制。

## 3. 技术方案
*   **引擎**: 基于 Canvas / WebGL 的轻量级渲染。
*   **宿主**: 微信小程序原生组件或 Webview。
*   **防作弊**: 关键逻辑服务端校验。
